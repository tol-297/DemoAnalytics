Чек-лист для починки, в зависимости от операционной системы:

### Вариант 1: Вы на Windows

В Windows LibreOffice поставляется со своим собственным интерпретатором Python. 
Команда `oooenv` должна модифицировать конфиг вашего окружения.

1. Убедитесь, что `venv` активирован (в консоли горит префикс окружения).
2. Выполните команду-тумблер:
```bash
oooenv env -t

```

3. Проверьте статус:
```bash
oooenv env -u

```
*(Команда должна вывести `True` или подтвердить, что окружение переключено на UNO).*
4. **Критичный шаг для Windows:** Если вы запускаете код из VS Code или PyCharm, после выполнения команды `oooenv env -t` **обязательно перезагрузите терминал или саму IDE**. Редакторы кэшируют пути окружения при запуске и могут не увидеть изменений "на лету".

### Вариант 2: Вы на Linux (Ubuntu/Debian)

В Linux `oooenv` пытается создать символические ссылки на системный мост UNO. Ошибка часто возникает, если сам системный пакет моста не установлен в ОС (он не всегда идет вместе с LibreOffice).

1. Откройте системный терминал (вне виртуального окружения) и доустановите мост:
```bash
sudo apt update
sudo apt install python3-uno

```

2. Вернитесь в свой проект, активируйте `venv` и сгенерируйте линки:
```bash
oooenv cmd-link -a

```

### Вариант 3: Вы на macOS

На маке `oooenv` тоже использует симлинки.
При активированном виртуальном окружении просто выполните:

```bash
oooenv cmd-link -a

```

### 🔍 Запасной план

Если команды выше не помогли, возможно, LibreOffice установлен в нестандартную папку (например, вы ставили его на диск `D:` вместо `C:` или используете портативную версию).
В этом случае `oooenv` не может найти исходники. Вы можете указать путь принудительно, добавив в систему переменную окружения `UDEV_LO_PATH` (или `LO_MAC_PATH` для macOS), указывающую на корневую папку LibreOffice.
